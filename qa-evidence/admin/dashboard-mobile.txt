# Admin Dashboard — Mobile (375×812)
# Captured 2026-04-22 by Claude during Chunk A QA walk.

## Visual observation (from preview_screenshot)

**Critical: tab bar overflows at 375px wide.** Labels visibly
overlap — "DASHBOARD" runs into "MARKETS" rendering as
"DASHBOARDARKETS". All 7 tab buttons are forced into a single
no-wrap row with no horizontal scroll indicator.

**Stat labels wrap awkwardly.** "ITEMS / WK →" breaks to two
lines, orphaning the arrow below "WK". Same for "SOLD / WK →".
The dashboard fix from commit 5393100 (adding arrows) works
but the container wasn't sized for the new width.

**Recent Actions list is raw event-type strings.** Shows
"send_dealer_blast" verbatim to an admin. No admin name,
no market context, no count. Not human-readable.

**No admin-mode branding.** The only indicator you're in /admin
is the tab bar itself. A subtle "ADMIN" chip near the logo or
a distinct background would help confirm the role at a glance.
